# MegamanRemix
Megaman Melies
A análise da viabilidade de continuar o projeto envolve diversos parâmetros externos ao jogo, principalmente a disponibilidade de recursos financeiros e humanos para o desenvolvimento e a quantidade de tempo para a entrega de uma versão funcional. Não disponíveis essas informações, resta avaliar o seu estado atual e projetar o grau de dificuldade envolvido na sua produção.
Em uma análise inicial, verifica-se que o projeto se encontra recém começado, com um mínimo de funções já implementadas. Grosso modo, no nível 1, as funções do jogador poderiam ser assim inventariadas: respawn, barra de vida, correr, pular simples e tiro simples. Em termos de cenário, há um corredor, três barris, uma torre no teto, um fosso de elevador, alguns andares vazios e um carro no topo. O nível 2 encontra-se em estágio ainda mais primário, consistindo em um carro e uma aranha mecânica de aspecto interessante, mas ainda desprovida de um propósito claro.
Os scripts mal são funcionais, com mecânicas básicas como o salto duplo e o deslizamento sequer tendo sido implementadas. As artes também são emprestadas, não se observando nada originalmente criado para o projeto.
Ainda mais preocupante é a falta de elementos mínimos para dar a dinâmica do jogo. Não se encontra presente uma forma de vencer, limitada a sua mecânica a correr, pular e atirar sem um propósito aparente. Há uma forma de morrer, pulando em direção à torre no teto repetidas vezes, mas que não encerra o jogo, reiniciando logo em seguida. O menu ainda não funciona, limitando-se a uma tela com três botões.
A conclusão do grupo é a de que o projeto se encontra tão incipiente que levá-lo adiante praticamente equivaleria a começá-lo do zero. Vale dizer, o que já foi feito não representa grande economia de tempo ou de recursos, de modo que, tomando-se em conta o projeto tal qual se encontra, não seria viável.
